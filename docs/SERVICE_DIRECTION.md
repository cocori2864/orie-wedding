# ORIÉ 서비스 방향성 및 기능 명세서

## 1. 서비스 정체성 (Identity)
**"단순한 쇼핑몰이 아닌, 하이엔드 웨딩 부케 아틀리에"**
- ORIÉ는 공산품을 판매하는 곳이 아니라, 작가의 작품(부케)을 전시하고 예약을 받는 공간입니다.
- 방문객에게 '구매'보다는 '감상'과 '의뢰'의 경험을 제공해야 합니다.

---

## 2. 핵심 기능 명세 (Core Features)

### A. 예약 슬롯 시스템 (Time-based Inventory)
**목적**: 핸드메이드 제작 특성상 발생하는 일일 생산량 한계(Capa) 관리 및 희소성 유지.

**구현 방안**:
1.  **슬롯 관리**: 날짜별 제작 가능 수량(Quota) 설정 (예: 평일 2개, 주말 5개).
2.  **예약 제한**:
    - 고객이 예식일(수령일) 선택 시, 해당 날짜의 잔여 슬롯 확인.
    - 슬롯 마감 시 해당 날짜 선택 불가 (Calendar UI에서 비활성화).
3.  **동시성 제어**: 결제 시점에 슬롯 잔여 여부를 재확인하여 오버부킹 방지.

### B. 분할 결제 시스템 (Split Payment)
**목적**: 예약금 우선 결제 후, 상담을 통해 확정된 최종 금액(잔금)을 결제하는 유연한 프로세스.

**프로세스**:
1.  **예약 단계 (Deposit)**:
    - 고객은 상담 예약을 위해 고정된 **예약금(예: 50,000원)**을 먼저 결제합니다.
    - 결제 수단: 계좌이체(별도의 결제수단 필요 없음).
2.  **상담 및 제작 (Consultation)**:
    - 작가와 상담 후 꽃 시장 시세, 추가 옵션 등을 반영하여 최종 금액 확정.
    - 관리자가 해당 주문 건에 대해 **'잔금(Balance)'** 금액을 입력.
3.  **잔금 결제 (Final Payment)**:
    - 고객에게 잔금 결제 알림 발송 (또는 마이페이지 확인).
    - 고객은 마이페이지에서 **[잔금 결제하기]** 버튼을 통해 차액을 카드로 결제.
    - *이 기능이 '개인 결제창' 역할을 수행함.*

## 3. UI/UX 방향성 (Design Direction)

### A. 메인 화면: 갤러리 (Gallery)
**컨셉**: "온라인 전시회"
- **카테고리 네비게이션 (Fixed Top Center)**:
    - 화면 상단 중앙에 **[촬영부케 | 본식부케 | 기타]** 카테고리를 고정 노출.
    - 사용자가 원하는 용도의 작품만 필터링하여 감상할 수 있도록 유도.
- **레이아웃**: 정형화된 그리드보다는 핀터레스트(Masonry) 스타일이나, 화면을 가득 채우는 풀스크린 슬라이더 활용.
- **정보 노출 최소화**:
    - 가격, 장바구니 버튼 등 상업적인 요소는 초기 화면에서 숨김.
    - 마우스 오버(Hover) 또는 클릭 시에만 작품명과 간략한 설명 노출.
- **용어 변경**:
    - Shop → **Collection / Works**
    - Product → **Piece / Objet**
    - Buy Now → **Inquire / Reserve**

### B. 상세 페이지: 도슨트 (Docent)
- 상품 스펙(사이즈, 소재) 나열보다는, 해당 부케의 **영감(Inspiration), 분위기, 어울리는 드레스** 등에 대한 스토리텔링 위주 구성.

## 4. 개발 로드맵 (Roadmap)
1.  **Phase 1 (비주얼 리뉴얼)**: 메인 페이지를 갤러리 형태로 전면 개편하여 브랜드 이미지 구축.
2.  **Phase 2 (예약 시스템)**: DB 스키마 변경(슬롯, 분할 결제) 및 예약 프로세스 구현.
3.  **Phase 3 (관리자 고도화)**: 일별 슬롯 관리 및 개별 주문 잔금 설정 기능 추가.
